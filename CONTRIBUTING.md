# Contributing to Mathematica to LaTeX Converter

Thank you for your interest in contributing to this project! 

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion for improvement:

1. Check if the issue already exists in the [issue tracker](https://github.com/Bradley-TaulUAH/mathematica-to-latex/issues)
2. If not, create a new issue with:
   - A clear, descriptive title
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Example Mathematica notebook file (if applicable)
   - Generated LaTeX output (if applicable)

### Submitting Changes

1. Fork the repository
2. Create a new branch for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Test your changes with various Mathematica notebooks
5. Commit your changes with clear, descriptive commit messages
6. Push to your fork
7. Submit a pull request

### Code Style

- Follow PEP 8 style guidelines for Python code
- Use descriptive variable and function names
- Add comments for complex logic
- Update documentation when adding new features

### Testing

Before submitting a pull request:

- Test the converter with various types of Mathematica notebooks
- Verify that generated LaTeX compiles correctly
- Check that existing functionality is not broken

## Questions?

Feel free to open an issue for questions or discussions about potential contributions.
